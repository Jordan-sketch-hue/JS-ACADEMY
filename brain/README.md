# AXIOM BRAIN — your own model, your own weights, no AI company in the loop

This folder is the exact learning process every large AI uses, mapped to operator
scale. Run it and you own the result: a neural network trained on code whose
weights live on YOUR disk and serve from YOUR Railway box. Nobody can revoke it,
meter it, or watch it.

## How every AI learns (the whole secret)

```
read code → hide the next token → predict it → measure the error
→ adjust the network's weights a tiny step → repeat, billions of times
```

That loop is called next-token prediction trained by gradient descent. It is
public science. The difference between this folder and Anthropic is not the
method — it is only scale (their run: months on tens of thousands of GPUs).

## The honest physics

| | Frontier lab | You (this pipeline) |
|---|---|---|
| Parameters | hundreds of billions | 0.5 – 1.5 billion |
| Training data | trillions of tokens | ~1–10B tokens of code (open datasets) |
| Hardware | GPU warehouse, months | 1 rented GPU, hours ($0.40–0.80/hr) |
| Result | general genius | **specialist you own** |

A 1.5B code model will not out-think a frontier model in general. But a small
model fine-tuned hard on ONE niche — your stack, your style, your patterns —
gets sharper *inside that niche* than a general model that spreads its capacity
over everything. Specialists beat generalists at the specialty. That, plus
total ownership, is the play.

## The pipeline (3 scripts)

1. **`dataset.py`** — builds your training corpus:
   - open code datasets (HTML/CSS/JS/React) from Hugging Face — free, legal
   - reference material (MDN/W3Schools-style docs you save into `corpus/`)
   - **your own AXIOM knowledge base** (`axiom-knowledge-base.json` export)
     so the model absorbs YOUR build history and style

2. **`train.py`** — the actual learning loop. Starts from an open-weights code
   model (Qwen2.5-Coder — Apache-2.0, free, yours to keep) and continues its
   training on your corpus with LoRA. This is literally the same
   next-token-prediction + gradient-descent loop the big labs run, on your data.
   - Rent one GPU (RTX 4090 / A10) on any per-hour service, or run on any
     24GB+ GPU you own. A first useful run: 2–6 hours.

3. **`serve.py`** — serves your trained weights as a tiny HTTP API. Deploy on
   your Railway (always-on lane) or any box. From that moment your model runs
   with zero external AI dependency — internet only needed to reach your own
   server.

## Wiring it into AXIOM

AXIOM's CODEGEN node checks for a brain endpoint. In the AXIOM browser console:

```js
localStorage.setItem("axiom_brain_url", "https://your-brain.up.railway.app");
```

From then on every directive is generated by YOUR model. If the brain is
unreachable, AXIOM falls back to its built-in templates — it never goes down.

## The growth loop (how it keeps getting smarter)

1. Use AXIOM daily — every build logs to the knowledge base
2. Export the KB, re-run `dataset.py` + `train.py` (a few $ per run)
3. Redeploy the new weights to Railway
4. Your model now knows what it learned from working for you

Each cycle the specialist gets deeper. That is identical in kind to how labs
improve models between versions — collect better data, train again.

## Run order

```bash
pip install -r requirements.txt
python dataset.py            # build corpus/train.jsonl
python train.py              # produces weights/ (yours forever)
python serve.py              # or deploy serve.py + weights/ to Railway
```
